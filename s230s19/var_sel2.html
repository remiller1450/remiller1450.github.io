Variable Selection and Regression (part 2)

Ryan Miller

Regression

  • In supervised learning we use \(\mathbf{X}\) to predict \(Y\) via: \(Y = f(\mathbf{X}) + \epsilon\)
  • For linear regression, \(f(\mathbf{X}) = \mathbf{X} \mathbf{\beta}\) and \(\epsilon_i \sim N(0,\sigma^2)\)
  • The linear regression model is fit (trained) by finding estimates \(\hat{\mathbf{\beta}}\) which minimize the residual sum of squares: \[RSS = \sum_i (y_i - \mathbf{x}_i^T \hat{\mathbf{\beta}})^2\]

Regression

  • Often \(\hat{\mathbf{\beta}}\) is a good, or even optimal, estimator of \(\mathbf{\beta}\)
  • However, in many situations, \(\hat{\mathbf{\beta}}\) is not a very good estimator. Examples include:
    • When the number of variables in \(\mathbf{X}\) is large
    • When the variables of \(\mathbf{X}\) are highly correlated
    • In these situations, \(\hat{\mathbf{\beta}}\) can be unstable (high variability)

Remedies - Subset Selection

  • One solution is not to use all of the variables \(\mathbf{X}\), selecting only a subset of variables to use in the model
  • Many algorithmic approaches exist to variable selection, the simplest is the best subsets approach
    • Fit models for all possible combinations of the predictor variables
    • Use a metric (adjusted \(R^2\), \(AIC\), \(BIC\), or cross-validation error) to select one of these models
    • Best subsets does not scale well when the number of predictors is large

Friedmans Benchmark Simulation

  • To explore variable selection, well introduce a well-known benchmark simulation setup:

\[y = 10sin(\pi x_1 x_2) + 20(x_3 - 0.5)^3 + 10x_4 + 5x_5 + N(0,1) \]

  • There are five additional predictors, \(x_6, \ldots, x_{10}\), that are unrelated with \(y\) (ie: true coefficients of zero)
  • All predictor variables, \(x_1, \ldots, x_{10}\), are uniformly distributed between 0 and 1
  • Friedmans data setup is useful because it contains:
    • Variables that interact with each other (\(x_1, x_2\))
    • A variable with a non-linear effect (\(x_3\))
    • Variables with different sized linear effects (\(x_4, x_5\))
    • Variables that contribute nothing (\(x_6, \ldots, x_{10}\))

Remedies - Stepwise Selection

  • A relatively modest number of predictors (ie: 10) leads to over 1,000 possible models
    • To make the best subsets approach doable, we needed to arbitrarily restrict ourselves to four varialbes
  • Most of these models arent promising candidates, so an alternative approach is to only check certain variable combinations that show promise
  • Stepwise selection algorithms are one of way of doing this
    • Forward stepwise algorithms start with an empty model and add (or later subtract) variables one at a time
    • Backward stepwise algorithms start with a full (saturated) model and substract (or later add) variables one at a time
  • Because these algorithms scale fairly well, we can apply them to the full Friedman data:
## Forward feature selection

Remedies - Principle Components Regression

  • Another alternative is to derive a smaller set of independent predictor variables using principle components
  • This is known as principle components regression:
    • Perform PCA
    • Fit a linear regression model using the first \(P\) principle components as predictor variables
df <- data.frame(y = sim$y, X)
pcreg.fit <- train(y ~ ., data = df, method = "pcr", trControl = fit.control, tuneGrid = data.frame(ncomp = 1:10), seed = 10)

## Alternatively, pre-process using PCA
set.seed(10)
pcpre.fit <- train(y ~ ., data = df, method = "lm", trControl = fit.control, preProcess = "pca")

resamps <- resamples(list(back = backward_subs, forward = forward_subs, pc = pcreg.fit, pc2 = pcpre.fit))

Remarks:

  • By default, pre-processing will select the number of components necessary to retain 95% of the variability in \(X\)
  • Using method = "pcr" will check various numbers of components (recall that resamps uses the optimal model produced by train)

Problems with these methods

  • Principle components regression can be effective, but it is difficult to interpret
  • Stepwise selection algorithms are greedy, they dont lead to globally optimal solutions
  • Best subsets regression doesnt scale well, its nearly impossible for even moderate \(p\) (like 10)

Advice

  • For most applications of linear regression (small-ish \(p\)), manual model building is preferable to relying on algorithms
  • Manual model building starts to become difficult when the number of available predictors becomes larger
    • Algorithms can provide a good starting point that can be further tweaked by the analyst
