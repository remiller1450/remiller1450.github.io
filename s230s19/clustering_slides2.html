  • Today we will explore a couple of other clustering algorithms, and in doing well see an approach to clustering categorical variables

k-Medoids?

  • In statistics, the mean is the most popular choice for summarizing a numeric variable
    • In some applications, the mean can be unduly influenced by outliers or skew, making it a poor representative of the variables center
  • A robust alternative to the mean is the median, so why not use the median to form cluster centers?

Partitioning Around Medoids (PAM)

  • PAM is a clustering algorithm that searches for \(k\) objects called medoids among the observed data-points
    • The algorithm starts with the medoids being chosen at random
    • Given a set of medoids, the other data-points assigned to the cluster defined by their nearest medoid
    • Then the algorithm performs a swapping step, where it checks for improvements by switching the medoids
    • If a medoid is swapped, the other data-points are re-assigned, and the swapping step is repeated
    • The prior two steps happen iteratively until a solution is reached

When to use PAM?

  • Because the cluster centers in PAM are data-points themselves, the algorithm only requires a distance matrix!
    • We will later see that we can use PAM on data containing categorical variables (after constructing a suitable distance matrix)
  • PAM is also more robust to outliers and skew than \(k\)-means, but its computation cost is greater and it can be more sensitive to the choice of initial cluster centers

Hierarchical Clustering

  • Both PAM and \(k\)-means are similar in nature, they iteratively consider partitions the data, re-evaluating a global objective function at each iteration
  • An alternative would be to greedily aggregate (or separate) data-points, creating a tree-like structure known as a dendrogram
  • The y-axis of the dendrogram is a measure of closeness
d <- get_dist(scale(USArrests))
hc <- hclust(d)

fviz_dend(hc, cex = 0.4, h = 2.5)

  • Iowa and New Hampshire are very close, you dont have to go very far before both states share a cluster
  • South Dakota and Iowa are modestly close, you have go to a height of ~2.5 to find a cluster containing both
  • Iowa and South Carolina are very far, they are only clustered together once all of the data are in the same cluster

Agglomerative vs.Divisive

  • There are two contrasting approaches to hierarchical clustering (ie: creating a dendrogram)
    • Agglomerative, or bottom-up approaches, which start with each data-point as its own cluster (leaf), and then combine similar clusters (branches) until all of the data-points are grouped together in single cluster (root)
    • Divisive, or top-down approaches, which start all of the data-points in a single cluster (root); this cluster is then divided into smaller clusters (branches) based upon dissimilarity, divisions continue until each data-point is its own cluster.
  • The acronyms AGNES (AGglomerative NESting) and DIANA (DIvisive ANAlysis) describe the most popular of these approaches
d <- get_dist(scale(USArrests))
ag <- agnes(d)
di <- diana(d)
hc <- hclust(d) ## hclust is another agglomerative clustering method

fviz_dend(ag, cex = 0.4) + labs(title = "DIANA")

fviz_dend(di, cex = 0.4) + labs(title = "AGNES")

fviz_dend(hc, cex = 0.4) + labs(title = "HCLUST")

Cutting Dendograms

  • A weakness of hierarchical clustering is the difficulty in choosing the number of clusters (where to cut the dendrogram)
    • Often, the number of clusters is determined by visual inspection
  • Some methods, like the average silhouette distance, are still possible but there is less consensus on how to use them
  • fviz_dend allows dendrograms to be cut by specifying a number of clusters k or by specifying a height h
fviz_dend(ag, k = 3, cex = 0.5)

## cutree cuts a dendrogram
sil_ag <- silhouette(cutree(ag, 4), dist = d)
rownames(sil_ag) <- rownames(USArrests)
fviz_silhouette(sil_ag, label = TRUE)

Clustering with Categorical Variables?

  • Clustering works well for numeric data because distance is well-defined
  • We can still apply clustering to categorical (or mixed numeric and categorical) data, but we need to define a more flexible distance measure
  • One such metric is Gower Distance, the idea is to use a distance metric that work well for type of variable under consideration, and re-scale it to range from 0 to 1 so that all variable types are on a level playing field

Gower Distance

  • For a pair of data-points, a and b, define:
    • \(d_{a,b} = \sum_{j = 1}^{p} h_j( x_{a,j}, x_{b,j})\)
    • \(h_j()\) depends on the type of variable \(j\)
    • For numeric variables, \(h_j(x_{a,j}, x_{b,j}) = \tfrac{|x_{a,j} - x_{b,j}|}{R_j}\), where \(R_j\) is the range of the \(j^{th}\) variable
    • For categorical variables, \(h_j(x_{a,j}, x_{b,j}) = 1\) if the \(x_{a,j}\) and \(x_{b,j}\) are the same category, and \(h_j(x_{a,j}, x_{b,j}) = 0\) otherwise

Using Gower Distance

  • Gower distance be used to construct a distance matrix, allowing it to be used with any clustering method that relies upon a dissimilarity (distance) matrix, such as:
    • PAM
    • DIANA
    • AGNES
    • Hclust
    • DBSCAN (not presented, but also popular)
