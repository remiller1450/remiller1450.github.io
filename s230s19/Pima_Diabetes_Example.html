spec <- specificity(tab) # Calculate test set specificity

print(data.frame(acc, sens, spec))

Accumulated Local Effects Plots

  • PDPs are easy to understand - they show the average marginal effect at particular value of a predictor variable across all of the data points
    • Unfortunately, they rely on unrealistic extrapolations when variables are correlated
    • Imagine correlated predictor variables “# bedrooms” and “living area”
    • The PDP effect for a living area of 800 sqft is found replacing the living areas of all data points with 800 while leaving their values for other variables unchanged
    • Plugging in such a small living area makes no sense for a house with 8 bedrooms
  • Accumulated Local Effects plots (ALEs) show the average conditional effect at a particular value of the predictor variable (estimated locally using a neighborhood)
    • The ALE for a living area of 800 sqft only uses houses with areas of about 800, estimating the living area effect via the difference in the predictions when pretending these houses are all 850 sqft versus pretending they are all 750 sqft
    • Differencing is intended to isolate the effect of variable of interest
    • ALE is centered, the value of the ALE can be interpreted as the main effect of the feature at a certain value compared to the average prediction of the data
model <- Predictor$new(model = rf.fit, data = test.data2)

## The grid size argument defines how many neighborhood
## We can see 5 line segments in this plot
e1 <- FeatureEffect$new(model, "glucose", method = "ale", grid.size = 5)
plot(e1) + theme_bw()

## Caution, not much data is in the top left!
e2 <- FeatureEffect$new(model, c("glucose", "insulin"), method = "ale")

The Next Steps/Caveats

  • If you were going to use these results real life decisions, you’d want to:
    • More careful consider the predictor variables; you might construct new derived variables or apply transformations
    • Revisit the missing data imputation; insulin was the most important predictor, and also among the most frequently missing
    • Spend more time looking at tuning parameter grids; the default tuning parameters caret chooses might not be optimal for this application
    • Spend time researching the model(s) that worked well on the training data before making a decision; caret is best used as a screening tool to help you filter lots of possible models in a systematic fashion
