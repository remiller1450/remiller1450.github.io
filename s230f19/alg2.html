Non-parametric methods

The goal in supervised learning is to model/predict an outcome variable \(Y\) using a set of inputs, to do so we adopt the following framework:

\[Y = f(\mathbf{X}) + \epsilon\]

  • Non-parametric methods, do not impose any particular structure on \(f\), instead applying an algorithm for generating an \(\hat{f}\)
    • These methods often provide high flexibility
    • We previously saw this with \(k\)-nearest neighbors and its tendency to overfit the data

Classification and Regression Trees (CART)

  • An extremely intuitive non-parametric approach to predicting any type of outcome is to apply a series of decision rules
    • For example, predict Yes if \(X_1\) is less than 5 and \(X_2\) is greater than zero, otherwise predict No
    • These rules can be organized into a tree-structure

How are these splits determined?

CART uses a procedure known as recursive binary splitting, you can view this as an algorithm:

  1. Starting with the full dataset, search for the splitting rule that maximizes accuracy in the resulting nodes
  2. Considering each node separately, search for another splitting rule that maximizes accuracy in the resulting sub-nodes
  3. Repeat until a suitable level of purity is reached (or until additional splits fail to sufficiciently improve accuracy)

For more information on splitting and stopping rules see: https://en.wikipedia.org/wiki/Decision_tree_learning or http://pages.stat.wisc.edu/~loh/treeprogs/guide/wires11.pdf

set.seed(123)
fit1 <- train(SalePrice ~ GrLivArea, data = houses, method = "knn", trControl = tctrl)

rs <- resamples(list(KNN = fit1, LM = fit2, CART = fit3))

Conclusion

  • Classification and regression trees are flexible and easy to interpret
  • Like k-nearest neighbors, their flexibility makes them prone towards overfitting
  • Soon well cover a black-box extension of trees known as random forests that effectively addresses this issue, allowing for excellent prediction
